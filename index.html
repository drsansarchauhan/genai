Syllabus & CO-PO Mapping
CSAI601 · 3-0-0 · B.Tech CS/IT · Semester VI/VII

Unit 1 – Foundations of Large Language Models (7 hrs)

History of AI and NLP: rule-based to statistical to neural. Word embeddings (Word2Vec, GloVe). Attention mechanism. Transformer architecture (encoder-decoder, multi-head attention, positional encoding). BERT, GPT lineage. Introduction to Claude: Anthropic's mission, Claude 3 family. Constitutional AI (CAI): principles, RLHF vs. RLAIF. Claude's key capabilities: long context, vision, code generation. Hands-on: first API call with Python.

Unit 2 – Prompt Engineering (8 hrs)

What is prompt engineering and why it matters. Zero-shot prompting. Few-shot prompting with examples. Chain-of-Thought (CoT) prompting. Role-playing and system prompts. Structured output (JSON, XML, Markdown). Context management: token limits, conversation history. Advanced: ReAct prompting, Tree of Thoughts, Self-consistency. Negative prompting and guardrails. Practical workshop: prompt optimisation for a real task.

Unit 3 – Building Applications with Claude API (8 hrs)

Claude API overview: authentication, models, rate limits. Python Anthropic SDK: messages API, streaming. Building a multi-turn chatbot. Tool use / function calling: schema definition, result passing. Retrieval-Augmented Generation (RAG): embeddings, vector databases, integration with Claude. Vision API: image inputs, multimodal prompts. Project lab: design and build a functional application end-to-end.

Unit 4 – Responsible AI, Safety & Ethics (6 hrs)

Bias in LLMs: sources, types, mitigation. Hallucinations: causes, detection, grounding strategies. Constitutional AI deep dive: principles list, RLAIF training loop. Privacy and data security in AI applications. Responsible Use Guidelines: Anthropic's policies, acceptable use. Case studies: AI misuse incidents, societal impact. AI governance frameworks: EU AI Act overview, responsible AI checklist.

Unit 5 – Advanced Topics & Capstone (7 hrs)

Agentic AI: agent loops, planning, memory, tools. Multi-agent architectures. Model Context Protocol (MCP): connectors, servers. Claude in production: deployment patterns, cost optimisation, latency. Fine-tuning concepts vs. prompt tuning vs. RAG. Industry applications: healthcare, legal, education, coding assistants. The future of LLMs: multimodality, reasoning, alignment. Capstone project presentations and peer review.

Unit 1 — Foundations of Large Language Models
7 Lecture Hours · Maps to CO1

📄 Open Lecture Notes 📊 Download Slides (PPTX)

Unit Learning Objectives

By the end of this unit, students will be able to: (1) trace the evolution of NLP from rule-based to transformer-based systems; (2) explain the multi-head self-attention mechanism and positional encoding; (3) describe the Constitutional AI training paradigm; (4) enumerate Claude's key capabilities; (5) make a basic API call to Claude using Python.

  • L01
    History of AI & NLP — From Rules to Neural Nets
    ELIZA, symbolic AI, statistical NLP (n-grams, HMMs), word embeddings (Word2Vec 2013, GloVe 2014), limitations of RNNs and LSTMs, the vanishing gradient problem. The "Attention is All You Need" moment (2017).
    Lecture
  • L02
    Deep Dive: Transformer Architecture
    Encoder-decoder structure. Self-attention: Query, Key, Value matrices. Multi-head attention. Positional encoding (sinusoidal). Layer normalisation. Feed-forward sub-layers. Complexity O(n²). BERT (encoder-only) vs GPT (decoder-only) vs T5 (seq2seq).
    Lecture
  • L03
    From GPT-3 to Claude — The LLM Landscape
    Scaling laws (Kaplan et al.). RLHF pipeline. ChatGPT breakthrough. Anthropic's founding story. Claude 1 → Claude 3 family (Haiku, Sonnet, Opus). Competitive landscape: GPT-4o, Gemini, Llama. Differentiation: safety, long context (200K tokens), honesty.
    Lecture
  • L04
    Constitutional AI — How Claude is Trained
    Reinforcement Learning from Human Feedback (RLHF): reward model, PPO. Constitutional AI (CAI): generating a "constitution" of principles. RLAIF: AI providing feedback instead of humans. HHH framework: Helpful, Harmless, Honest. Red-teaming and safety evaluation.
    Lecture
  • L05
    Claude's Capabilities — Text, Vision, Code
    Long context window (200K tokens): use-cases and challenges. Native multimodality: image understanding, chart analysis. Code generation and execution. Tool use. Multilingual capability. Context window management strategies. Tokenisation: BPE, SentencePiece.
    Lecture
  • L06
    Tokenisation, Temperature & Sampling Strategies
    Byte-Pair Encoding (BPE). Token counting. Temperature, top-p, top-k sampling. Greedy vs beam search. Stop sequences. Max tokens. How these parameters shape output quality. Practical demonstration.
    Lecture
  • L07
    Lab — First API Call & Exploring the Playground
    Setting up Anthropic API key. Python SDK install (pip install anthropic). client.messages.create(). Passing system + user messages. Inspecting response object. Counting tokens. Exploring claude.ai playground. Live demo: build a one-function summariser.
    Lab

Key Concepts Summary

ConceptOne-line Definition
Self-AttentionMechanism allowing each token to attend to all other tokens weighted by relevance.
RLHFTraining technique using human preference labels to shape model behaviour via a reward model.
Constitutional AIAnthropic's method of using a set of written principles + AI feedback to make models safer without constant human labelling.
TemperatureControls randomness in sampling; 0 = deterministic, 1 = standard, >1 = creative/random.
BPEByte-Pair Encoding — sub-word tokenisation that balances vocabulary size and coverage.
Long ContextClaude supports up to 200,000 tokens in a single context window, enabling full-document analysis.

Unit 2 — Prompt Engineering
8 Lecture Hours · Maps to CO2

Unit Learning Objectives

Students will: (1) distinguish between zero-shot, few-shot, and chain-of-thought prompting; (2) design system prompts and role-play personas for specialised tasks; (3) produce structured outputs (JSON, XML) through prompt design; (4) apply ReAct and Tree-of-Thoughts strategies; (5) systematically optimise prompts using iteration and evaluation.

  • L08
    Introduction to Prompt Engineering
    Why prompting matters. The prompt-as-program metaphor. Anatomy of a prompt: instruction, context, input, output format. Prompt vs fine-tuning trade-offs. Anthropic's prompt engineering guide overview.
    Lecture
  • L09
    Zero-shot & Few-shot Prompting
    Zero-shot: relying purely on instruction. Few-shot: in-context examples as implicit training. Selecting and ordering examples. Label formatting. Sensitivity to example quality. Brown et al. (2020) GPT-3 findings. Hands-on: classification task.
    Lecture
  • L10
    Chain-of-Thought Prompting
    Wei et al. (2022): "Let's think step by step." Manual CoT vs auto CoT. Zero-shot CoT. Self-consistency (multiple reasoning paths → majority vote). Program-of-thought prompting. When CoT helps and when it doesn't. Hands-on: multi-step math and logic.
    Lecture
  • L11
    System Prompts, Role-Playing & Personas
    The system/user/assistant turn structure. Writing effective system prompts for Claude. Persona assignment. Tone and style control. Guardrails within system prompts. Context injection. Practical: build a domain expert assistant.
    Lecture
  • L12
    Structured Outputs — JSON, XML, Markdown
    Requesting JSON-formatted responses. Schema specification in prompts. Claude's native XML affinity. Markdown output for documents. Output validation and error handling. Using Claude for data extraction pipelines. Hands-on: extract structured data from unstructured text.
    Lecture
  • L13
    Context Management & Memory Strategies
    Token limit arithmetic. Summarisation strategies. Sliding window context. External memory with embeddings. Injecting retrieved context. Managing conversation history in multi-turn applications. Cost vs context length trade-offs.
    Lecture
  • L14
    Advanced Techniques — ReAct, ToT, Metacognition
    ReAct framework: Reasoning + Acting in interleaved steps. Tree of Thoughts: exploring multiple reasoning branches. Reflection and self-critique prompting. Meta-prompting: asking the model to improve its own prompt. Negative prompting and constraint injection.
    Lecture
  • L15
    Workshop — Prompt Optimisation Lab
    Iterative prompt refinement process. Evaluation rubrics for prompt output. A/B testing prompts. Prompt debugging: identifying failure modes. Group activity: each team optimises a prompt for a given task and presents results.
    Lab
Unit 3 — Building Applications with Claude API
8 Lecture Hours · Maps to CO3

Unit Learning Objectives

Students will: (1) configure and use the Anthropic Python SDK; (2) implement a multi-turn chatbot; (3) integrate tool/function calling; (4) build a basic RAG pipeline; (5) process image inputs via the Vision API; (6) deploy a mini application end-to-end.

  • L16
    Claude API Architecture & Authentication
    REST API structure. API keys, environment variables, rate limits. Available models (claude-3-5-sonnet, claude-3-haiku, etc.). Pricing model: input + output tokens. SDK vs raw HTTP. Error types: rate limit, context length, content policy. Best practices for key management.
    Lecture
  • L17
    Python SDK Deep Dive — Messages API
    anthropic.Anthropic() client. messages.create() parameters. Streaming responses with stream=True. Async SDK usage (AsyncAnthropic). Response object structure: stop_reason, usage, content. Handling multi-content blocks. Live coding session.
    Lab
  • L18
    Building a Multi-turn Chatbot
    Conversation history as a list of messages. Appending user/assistant turns. System prompt design. Stateless vs stateful architectures. Flask/FastAPI backend integration. Simple web front-end. Streaming UI with Server-Sent Events. Hands-on: build a domain-specific assistant.
    Lab
  • L19
    Tool Use & Function Calling
    Tool definition schema (JSON). Passing tools to the API. Claude's tool_use content blocks. Executing tool calls and returning results. Multi-tool agents. Practical use cases: web search, calculator, database query. Error handling in tool loops.
    Lecture
  • L20
    Retrieval-Augmented Generation (RAG)
    RAG architecture: indexing, retrieval, generation. Text chunking strategies. Embeddings (OpenAI, HuggingFace Sentence Transformers). Vector databases: Chroma, FAISS, Pinecone. Retrieval: cosine similarity, MMR. Injecting retrieved docs into Claude context. Evaluation: faithfulness, relevance.
    Lecture
  • L21
    Vision API — Multimodal Inputs
    Passing images: base64 encoding vs URL reference. Supported formats (JPEG, PNG, GIF, WebP). Image size limits and token costs. Use cases: document parsing, chart analysis, OCR-free data extraction, UI screenshot analysis. Multi-image prompts. Hands-on: build an image Q&A tool.
    Lab
  • L22
    Production Considerations — Caching, Streaming, Cost
    Prompt caching API (beta): reduce costs on repeated context. Batch API for async workloads. Latency optimisation: streaming, model selection. Cost estimation formula. Monitoring with logging middleware. Error retry strategies (exponential backoff). Testing LLM applications: deterministic vs non-deterministic.
    Lecture
  • L23
    Project Lab — End-to-End Application Build
    Students build a complete application in groups. Options: (a) PDF Q&A RAG system, (b) multi-tool research assistant, (c) code review bot. Peer presentation (5 min each). Evaluation criteria: functionality, code quality, prompt design, error handling.
    Lab
Unit 4 — Responsible AI, Safety & Ethics
6 Lecture Hours · Maps to CO4

Unit Learning Objectives

Students will: (1) identify sources and types of bias in LLMs; (2) explain why LLMs hallucinate and apply grounding strategies; (3) analyse Constitutional AI as a safety mechanism; (4) apply privacy and data security principles to AI apps; (5) evaluate AI systems against ethical frameworks.

  • L24
    Bias in LLMs — Sources, Types & Mitigation
    Training data bias: historical, representation, measurement. Social biases: gender, race, religion. Amplification through RLHF. Evaluation benchmarks: WinoBias, BBQ, TruthfulQA. Mitigation: data curation, adversarial training, post-hoc filtering. Anthropic's internal red-teaming process.
    Lecture
  • L25
    Hallucinations — Causes, Detection & Grounding
    Definition and taxonomy of hallucinations (intrinsic vs extrinsic). Why models confabulate: training objective mismatch. Detection techniques: self-consistency checking, chain-of-verification (CoVe). Grounding strategies: RAG, tool use, source citation. Building verifiable AI pipelines.
    Lecture
  • L26
    Constitutional AI Deep Dive & Governance
    CAI paper walkthrough. The 58-principle constitution. RLAIF training loop diagram. Comparison with OpenAI's approach. EU AI Act: risk categories, high-risk system requirements. UNESCO AI Ethics framework. Responsible AI maturity model. Industry self-regulation vs regulation.
    Lecture
  • L27
    Privacy, Security & Data Protection in AI
    PII in training data: risks and examples. Membership inference attacks. Prompt injection attacks. Data retention policies. GDPR implications for AI products. Designing privacy-preserving AI pipelines: anonymisation, differential privacy concepts. Secure API key management.
    Lecture
  • L28
    Responsible Use Guidelines & Acceptable Use
    Anthropic's Acceptable Use Policy. Prohibited uses: CSAM, weapon design, mass surveillance. Content moderation approaches. Designing safe applications: output filtering, rate limiting, logging. Responsible disclosure. Building AI for accessibility and equity. Developer responsibilities.
    Lecture
  • L29
    Case Studies & Ethical Discussion Seminar
    Case 1: Bing/Sydney persona incidents. Case 2: AI in hiring — Amazon's scrapped tool. Case 3: Medical AI misdiagnosis risks. Case 4: Deepfakes and disinformation. Case 5: Claude refusing harmful requests. Socratic discussion. Students present an AI ethics scenario analysis (group activity).
    Discussion

  • c) CAI uses AI-generated feedback based on a set of principles instead of human preference labels

5. [Short Answer, 2–3 lines] What is the HHH framework and why is it important for Claude?
  • Model Answer: HHH stands for Helpful, Harmless, Honest. It defines Claude's three core objectives: maximally helping users, avoiding harmful outputs, and being truthful. It is important because it operationalises Anthropic's mission of safe and beneficial AI into concrete training objectives.

Quiz 2 — Prompt Engineering

Unit 2 · 2 Marks · 15 min
1. "Let's think step by step" is associated with which prompting technique?
  • a) Few-shot prompting
  • b) Zero-shot Chain-of-Thought
  • c) Self-consistency
  • d) ReAct prompting
2. Self-consistency prompting improves accuracy by:
  • a) Using a larger model for each query
  • b) Sampling multiple diverse reasoning chains and taking the majority answer
  • c) Asking the model to critique its own output once
  • d) Adding more examples to the prompt
3. In ReAct prompting, "Action" refers to:
  • a) The model's final answer
  • b) Human feedback on the model output
  • c) A tool call or external operation triggered by the model
  • d) Rewriting the original prompt
4. Which of the following is NOT a benefit of structured output prompting?
  • a) Easier downstream parsing
  • b) Consistent response format
  • c) Better integration with APIs
  • d) Guaranteed reduction in hallucinations
5. [Short Answer] Explain the difference between a system prompt and a user prompt in Claude's conversation structure.
  • Model Answer: A system prompt is set by the developer/operator before the conversation and defines Claude's persona, task, constraints, and context. A user prompt is the actual message sent by the end-user during the conversation. System prompts take precedence and frame how Claude should interpret all subsequent user messages.
